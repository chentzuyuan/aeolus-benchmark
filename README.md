# Aeolus Benchmark: GBDT vs Deep Learning on Flight Delay Prediction

Reproducible benchmark comparing four tree-based (GBDT) models against the seven deep-learning models evaluated in [Xu et al. (2025) — *Aeolus*](https://arxiv.org/abs/2510.26616) on U.S. domestic flight delay prediction.

## Background

Xu et al. (2025) benchmark seven deep-learning architectures (MLP, ResNet, AutoInt, Tangos, TabulaRNN, etc.) on a 22-feature pre-departure flight delay dataset but do **not** include any GBDT baseline. This repository fills that gap and shows that all four GBDT models comfortably outperform all seven DL models, consistent with Grinsztajn et al. (2022) who demonstrate GBDT superiority on tabular data.

> **Note on Aeolus AUC methodology**: Xu et al. compute their reported AUC values from binary `predict()` outputs rather than probability scores, yielding a single operating-point estimate rather than standard AUC-ROC. We use probability-score AUC-ROC throughout to ensure methodological consistency.

## Dataset

- **Source**: BTS On-Time Performance (public) — June 1–15, 2024
- **Flights**: 287,845 domestic U.S. flights
- **Features**: 22 pre-departure features (matching Aeolus)
- **Split**: 6:2:2 chronological (train / val / test)
- **Tasks**: `ARR_Delay` (arrival delay ≥ 15 min) and `DEP_Delay` (departure delay ≥ 15 min)
- **Input file**: `aeolus_feat.parquet` (generated by upstream preprocessing)

## Results

### ARR\_Delay (Test AUC-ROC)

| Model family | Model | AUC |
|---|---|---|
| **Tree-based (ours)** | XGBoost (Optuna) | **0.6884** |
| Tree-based (ours) | Random Forest | 0.6867 |
| Tree-based (ours) | CatBoost (default) | 0.6829 |
| Tree-based (ours) | XGBoost (default) | 0.6817 |
| DL — Xu et al. (2025) | Best: AutoInt | ~0.623 |
| DL — Xu et al. (2025) | Other 6 models | < 0.623 |

**GBDT advantage: +0.059 to +0.065 over best DL**

### DEP\_Delay (Test AUC-ROC)

| Model family | Model | AUC |
|---|---|---|
| **Tree-based (ours)** | XGBoost (Optuna) | **0.6973** |
| Tree-based (ours) | Random Forest | 0.6954 |
| Tree-based (ours) | XGBoost (default) | 0.6936 |
| Tree-based (ours) | CatBoost (default) | 0.6920 |
| DL — Xu et al. (2025) | Best: MLP/Tangos/TabulaRNN | ~0.627 |
| DL — Xu et al. (2025) | Other 4 models | < 0.627 |

**GBDT advantage: +0.065 to +0.070 over best DL**

All full results (including reimplemented DL models) are stored in `checkpoint.json`.

## Repository Structure

```
aeolus_benchmark/
├── gbdt.py          # Phase 1: RF, XGBoost (default), CatBoost, XGBoost (Optuna)
├── dl.py            # Phase 2: MLP, ResNet, AutoInt (Aeolus setup + our setup)
├── run.sh           # One-click runner (gbdt → dl, with checkpointing)
├── checkpoint.json  # Final results for all 26 model × task combinations
└── logs/            # Training logs (gbdt.log, dl.log)
```

## Usage

```bash
# Install dependencies
pip install scikit-learn xgboost catboost optuna torch pandas pyarrow

# Run everything (GBDT first, then DL — checkpoint allows resumption)
bash run.sh

# Or run phases separately
python gbdt.py          # GBDT only
python dl.py            # DL only (reads checkpoint written by gbdt.py)
```

The scripts skip already-completed models by reading `checkpoint.json`, so a crashed run can be resumed safely.

## Environment

- Python 3.10+
- macOS Apple Silicon: GBDT and DL are run in **separate processes** to avoid the XGBoost + PyTorch SIGSEGV (known segfault on MPS/ARM)
- DL training uses CPU (`torch.device('cpu')`) for stability; MPS was disabled due to OOM kills
